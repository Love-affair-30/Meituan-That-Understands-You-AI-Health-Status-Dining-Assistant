# 懂你的美团：AI健康状态点餐助手

> 基于身体状态与饮食禁忌的 AI 外卖点餐助手

## 项目简介

“懂你的美团：AI健康状态点餐助手”面向身体不适、饮食忌口以及“今天吃什么”的决策难题。用户只需选择当前身体状态、想吃的类型和过敏忌口，系统便会分析饮食需求，过滤存在食材风险的菜品，并提供更适合当前状态的外卖推荐。

项目已经打通“状态填写 → AI 分析 → 菜品推荐 → 商品详情 → 购物车 → 确认订单”的完整点餐流程，让用户点得更省心、吃得更安心。

## 解决的问题

- 身体不舒服时，不清楚哪些餐食更合适。
- 外卖平台菜品繁多，筛选成本高，容易陷入选择困难。
- 普通关键词推荐不能可靠处理过敏、忌口等硬性限制。
- 菜名、图片、配料和推荐理由缺少一致性，影响用户判断。
- 健康建议与实际下单流程割裂，用户仍需重复搜索和筛选。

## 核心功能

### 1. 身体状态识别

支持嗓子疼、感冒发烧、痛经、胃不舒服、熬夜疲惫、酒后恢复等常见状态，并结合不适程度、饮食偏好和避开项生成饮食分析。

### 2. 忌口硬过滤

系统将鸡蛋、海鲜、牛奶、坚果、香菜等忌口设为硬性约束，而不是普通推荐偏好。

推荐前会同时检查：

- 菜品名称
- 主要食材
- 推荐理由
- 用户评价关键词
- 同类食材表达，如“蛋羹”“蒸蛋”“鱼虾蟹贝”等

若发现冲突菜品，系统会直接排除，不使用风险菜品补足推荐数量。

### 3. 个性化外卖推荐

系统结合当前状态，从清淡程度、温度、刺激性、营养结构、评分和配送时间等维度推荐餐品，并提供：

- 推荐理由
- 主要食材
- 评论关键词
- 图文一致度
- 商品价格和配送信息

### 4. 商品详情

点击推荐卡片的“+ 加入”可进入对应商品详情页，查看菜品大图、价格、门店、配送、食材、评价和 AI 适配理由。

### 5. 购物车编辑

加入购物车后可以：

- 查看当前已购商品
- 勾选或取消勾选商品
- 增加或减少数量
- 删除单个商品
- 全选或清空购物车
- 实时计算已选商品金额

### 6. 确认订单与结算

结算页展示配送地址、配送时间、商品清单、包装费、配送费、健康餐优惠、备注、餐具和支付方式，形成完整的外卖下单体验。

### 7. 状态周报

通过状态日历、推荐命中率和饮食提醒，帮助用户了解近期饮食选择与身体状态的关系。

## 产品流程

```text
选择身体状态
      ↓
补充程度、偏好与忌口
      ↓
AI 生成饮食分析
      ↓
过敏原与忌口硬过滤
      ↓
生成个性化外卖推荐
      ↓
查看商品详情
      ↓
编辑购物车
      ↓
确认订单并结算
```

## 技术实现

本项目为轻量级移动端交互原型，无需安装前端框架。

- HTML5：页面结构与语义化内容
- CSS3：移动端布局、卡片、抽屉、过渡动画及响应式样式
- 原生 JavaScript：状态管理、推荐过滤、页面导航、购物车和金额计算
- 本地图片资源：保证菜品图片稳定加载，并按菜品类型进行匹配

推荐逻辑采用“状态目录 + 商品元数据 + 过敏原规则”的方式实现。先根据身体状态生成候选集合，再执行忌口硬过滤，最后按综合推荐、配送时长、评分或少糖等条件展示结果。

## 本地运行

项目无需构建，可以直接打开 `index.html`。为避免浏览器对本地资源的限制，建议启动静态服务器：

```powershell
python -m http.server 4173
```

然后访问：

```text
http://127.0.0.1:4173
```

也可以使用 VS Code 的 Live Server 等静态服务器工具。

## 演示建议

1. 从首页选择“嗓子疼”进入状态点餐助手。
2. 在“过敏 / 忌口”中选择“鸡蛋”。
3. 查看 AI 分析，确认建议中不再出现蒸蛋或蛋羹。
4. 生成推荐，观察系统的忌口过滤提示。
5. 进入任一商品详情并加入购物车。
6. 打开购物车，修改数量和勾选状态。
7. 点击“去结算”，展示完整确认订单页面。

## 项目亮点

- 将身体状态转换为可执行的点餐建议，而非停留在健康知识展示。
- 将过敏和忌口设为硬约束，避免推荐逻辑与用户安全需求冲突。
- 推荐理由、主要食材和商品图片互相对应，降低判断成本。
- 完整覆盖推荐、选购、购物车和结算流程，具备较强的产品落地感。
- 采用纯前端实现，部署简单，适合快速演示与后续扩展。
